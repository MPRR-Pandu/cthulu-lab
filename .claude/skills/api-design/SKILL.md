# API Design

1. Understand domain and use cases
2. Define resources and relationships
3. Design endpoints following REST conventions
4. Define request/response schemas and error handling

Rules: nouns for resources (`/users` not `/getUsers`). Proper HTTP methods. Version APIs (`/api/v1/`). Consistent error format. Paginate list endpoints. ISO 8601 dates. Rate limit all endpoints.
