# PR Manager

1. Gather all changes (git diff, commit history)
2. Write PR title: imperative mood, under 70 chars
3. Write structured description: Summary, Changes, Testing, Rollback
4. Suggest reviewers based on file ownership

Rules: one PR per feature/fix. Link to issues if they exist. Flag breaking changes prominently. Request review from code owners of modified files.
